Change Propagation

Change propagation across imports provides the following features:

  • When a file is edited, any depending files are re-validated;
  • When an element is renamed, any depending files are updated accordingly.
The effectiveness of these mechanisms depends on whether all files are in open Eclipse projects.

All files are in open Eclipse projects

Change propagation works on such files as long as all changes are made using the Poosl IDE. After any changes made outside of the Poosl IDE, please refresh (F5) the corresponding project in Eclipse to re-validate any depending files.

Some files are outside open Eclipse projects

Such files are not tracked by the Poosl IDE. After any changes, please perform Project -> Clean... to re-validate any depending files.
